# cryptography_2

## Реалізація Струмок-256 та Струмок-512

Специфікація:

- Струмок-256: ключ 256 біт, ІV 256 біт, внутрішній стан 512 біт
- Струмок-512: ключ 512 біт, ІV 512 біт, внутрішній стан 1024 біт

`Внутрішній стан:
s[0..15] - LFSR (лінійний зсувний регістр, 16 × 64 біт)
r1, r2 - регістри нелінійної частини (FSM)`

`Параметри:
nwords - кількість 64-бітних слів для ключа та ІV
(4- Струмок-256, 8 -Струмок-512)`

### Завантаження ключа та ІV у внутрішній стан LFSR

Схема заповнення для Струмок-256 (nwords=4):
s[0..3] ← key[0..3]
s[4..7] ← iv[0..3]
s[8..11] ← key[0..3] (повторення)
s[12..15] ← iv[0..3] (повторення)

Для Струмок-512 (nwords=8):
s[0..7] ← key[0..7]
s[8..15] ← iv[0..7]

## Структура алгоритму:

- Лінійний зсувний регістр (LFSR): 16 слів по 64 біти
- Нелінійна частина (FSM): 2 слова-регістри (R1, R2)
- Нелінійне перетворення T (S-box + матриця MDS через попередньо обчислені таблиці)
- Множення на елементи α та α^-1 скінченного поля GF(2^64)

Крок LFSR відповідно до специфікації ДСТУ 8845:2019 :

    Характеристичний поліном над GF(2^64):
        s[16] = α·s[0] ⊕ s[2] ⊕ α⁻¹·s[15]
            = alpha_mul(s[0]) XOR s[2] XOR alphainv_mul(s[15])

    Після обчислення нового елемента стан зсувається:
        s[0..14] ← s[1..15], s[15] ← нове значення
        (deepseek написав красиво формули)

В реалізації коду біти переведено в байти для зручності використання обрахунків.

### Тестування реалізації шифру «Струмок»

1. (α та α⁻¹ — взаємно обернені)
2. Відтворюваність (однаковий ключ/ІV -> однакова гама)
3. Унікальність (різні ключі -> різні гами)
4. Симетрія (encrypt(decrypt(x)) == x)
5. Струмок-256 та Струмок-512 відрізняються для різних ключів
6. Референсні вектори (внутрішньо погоджені - для перевірки регресій)
7. Базова функціональність encrypt/decrypt

### Відомості про атаку часткового вгадування

Заявленою оцінкою стійкості Алгоритму Струмок-512 є $2^{512}$, але дана атака дозволяє зменшити її до $2^{448}$,
за умови, що зловмисник має перехоплену гаму. Атака проводиться таким чином:

1. Знаходження рівнянь функціонального зв'язку між словами шифру.
2. Вимірювання кількості потрібних вгадувань (Загальна кількість рівнянь = кількість змінних початкового стану = 18. Кількість змінних, які треба вгадати = 18 - кількість гам = 18-11 = 7, отже складність вгадування $(2^{64})^{7}$ = $2^{448}$).
3. Вгадування даних базисів (У програмному виконанні завдання дістав їх з алгоритму для спрощення).
4. Ітеративний пошук інших 44 змінних з відомих 18.

#### Практичне використання:
1. Дана атака може використовуватись у підборі людських паролів, оскільки, якщо ти вже знаєш вік людини, дату народження, ім'я тощо, ти маєш можливість створити алгоритм, що значно швидше перебирає усі реалістично можливі паролі.
2. Атака часткового вгадування могла бути використана на старий стандарт шифрування Bluetooth, де були використані 4 регістри різної довжини. Схема вгадувала 3 короткі, а четверту, яка була найдовша - обраховувала спираючись на 3 інші. Результат - стійкість падає з $2^{128}$ до $2^{38}$.
3. До WPA2 у шифруванні Wifi був стандарт TKIP, яким мав вразливість пов'язану з видачами помилок. При введені правильного байта система міняла природу помилки, отже можна було перебирати по одному байту, поки не отримаєш повний ключ. 

### Результат пошуку базисів та приклад знаходження інших змінних використовуючи ці базиси

Після використання пошуку базисів, були отримані такі базиси: s_16, s_12, s_15, r1_8, s_24, r2_10, r1_10.

Першою змінною, яка була знайдена з базисів стала r2_9 = 5691249495381789969, яка була знайдена з r1_8 = 5691249495381789969 за функціональним зв'язком r2_i+1 = r1_i.
Наступним цікавим прикладом можна взяти s_11 = 10399809127588421329, що був знайдений з r1_10 = 16044331019324593088 та r2_9 = 5691249495381789969, який був знайдений у минулому прикладі, методом XOR. 

Таким чином, ітеративно, алгоритм знаходить усі 62 змінні Струмка.
